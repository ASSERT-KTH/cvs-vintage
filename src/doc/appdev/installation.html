2. INSTALLATION

2.1 Java Development Kit

Tomcat will operate under any Java Development Kit (JDK) environment that provides a JDK 1.1 or JDK 1.2 (also known as Java2 Standard Edition, or J2SE) compatible platform. You will need a Java Development Kit, as opposed to a Java Runtime Environment, so that your servlets, other classes, and JSP pages can be compiled.

Compatible JDKs for many platforms (or links to where they can be found) are available at http://java.sun.com/jdk/.

The shell command line examples in this manual assume that you have set the JAVA_HOME environment variable to point at the base directory where you have installed the JDK (such as /usr/local/jdk1.2.2), have added the $JAVA_HOME/bin directory to your PATH environment variable (so that the compiler and other executable components can be located), and have initialized the CLASSPATH environment variable if necessary.

2.2 Jakarta Tomcat

Binary downloads of the Tomcat server are available from the http://jakarta.apache.org/downloads/binindex.html page. This manual assumes you are using the most recent milestone, beta, or release of Tomcat 3.1 -- many of the deployment details have changed since Tomcat 3.0 was released, although the general concepts are very similar.

The shell command line examples in this manual assume that you have set the TOMCAT_HOME environment variable to point at the base directory where you have installed Tomcat (such as /usr/local/tomcat), and have added the $TOMCAT_HOME/bin directory to your PATH environment variable. No adjustment to the CLASSPATH environment variable is normally required, unless your application requires additional Java classes or JAR files to be added to the system classpath.

2.3 Jakarta Ant

Binary downloads of the Ant development tool are available from the http://jakarta.apache.org/downloads/binindex.html page. This manual assumes you are using the most recent milestone, beta, or release of Tomcat 3.1 -- many of the deployment details have changed since Tomcat 3.0 was released, although the general concepts are very similar.

The shell command line examples in this manual assume that you have set the ANT_HOME environment variable to point at the base directory where you have installed Ant (such as /usr/local/ant), and have added the $ANT_HOME/bin directory to your PATH environment variable. No adjustment to the CLASSPATH environment variable is normally required.

2.4 Concurrent Version System (CVS)

Besides the required tools described above, you are strongly encouraged to download and install a source code control system, such as CVS, to maintain historical versions of the source files that make up your web application. Besides the server, you will also need appropriate client tools to check out source code files, and check in modified versions.

Detailed instructions for installing and using source code control applications is beyond the scope of this manual. However, CVS server and client tools for many platforms (along with documentation) can be downloaded from http://www.sourcegear.com/CVS.
