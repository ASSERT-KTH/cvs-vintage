Status of the Scarab Bug Tracking System (SBTS)

Last Updated: $Id: project_status.html,v 1.1 2000/12/18 05:03:40 jon Exp $

Currently this project is in its infancy. There has been a lot of interest in helping out with this project as well as a lot of feedback on the types of tracking and the ways to perform the tracking. All of this feedback is crucial towards generating a product that is usable across a wide audience and is appreciated. Please refer to the homepage of the project for the primary goals.

An outline of the current status of the project is described below...

Direction and Status...

The first steps that need to be taken in order to ensure the success of this project are (in order):

  • A database schema must be designed and agreed upon
  • The UI mockup needs to be completed and agreed upon
  • Actual implementation must proceed

The database schema is available in CVS and an image is available.

The IA/UI designs are currently in progress and will be available for review soon.

Technologies and Design...

Here is a list of technologies that are currently slated to be used within the project:

  • Turbine will be used as the underlying framework
  • Peer model and Torque within Turbine will provide the database abstraction
  • Localization Service within Turbine will be used so that the application can be used in any language
  • Webmacro will be used as the abstraction between the user interface and the business logic. We intend to make the look and feel of this application easily customizable.
  • MySQL will be the SQL backend.
  • Embarcadero ERStudio - SQL database design
  • Apache JServ and Tomcat will be the target Java Servlet engines, but this application should run on any JSDK 2.0 and higher Java Servlet engine.

The Scarab Design Document explains in detail what is expected of the application as well as our reasoning behind our choices.

What you can do to help...

All discussion of all parts of this application should go on the developers mailing list. That way, the largest amount of people can participate.

We will be maintaining a Scarab Tasks document that will provide areas where you can take charge and do development.
