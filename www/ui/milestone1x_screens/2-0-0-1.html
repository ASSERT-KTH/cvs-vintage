An artifact type is a unique instance of a Scarab database, including the following attributes:
  • a customizable set of attributes
  • a customizable set of options (per attribute)
  • a unique set of permissions per role

Once a user selects an artifact type, Scarab's navigation will indicate their selection state by dropping down a list of functions within that artifact type, giving authorized user's access to functionality such as:

  • enter a new issue
  • query the database
  • run reports on database activity

The user's input remains confined to the selected artifact type until they select another artifact type from the left side-bar navigation, or from the artifact type index.
