Reports screen index:

  1. Confine dataset - user limits the base set of data from which the report will be drawn
  2. Report data groups - User can group related options together into sub-groups - e.g. user selects
    Why confine the dataset and create date groups?: This functionality allow a third axis to be added to each intersection;
    e.g.
    • row= John Smith
    • col= p1
    • confining attribute (group)=Supported Mac OS
      • Mac 9.1
      • Mac 9.2
      • OSX
  3. Select/add row content - User selects the type of row content (e.g. attributes / options or Users / User attributes)
  4. Select/add column content - this index breaks down the three options for column content
  5. Output - At any point while building a report (e.g. selecting / adding column or row content), user can select to "Recalculate report". User can then review output, and then choose to either print, save report, or continue editing
  6. Save report - user enters this information if they choose to save report
  7. Saved reports index

Reports functionality notes:

  1. It is necessary to use the save button to save what you have selected - if you navigate with the tabs without saving, you lose your data.
  2. It is necessary to use the save button to save what you have selected - if you navigate with the tabs without saving, you lose your data (this only applies to selecting attributes / options)
  3. Data groups generate side by side comparisons per group (e.g. columns repeat per group per selected attributes / options or Users / User attributes
