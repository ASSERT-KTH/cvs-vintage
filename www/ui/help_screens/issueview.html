Querying issues:

3 ways to query for issues:

  • advanced- allows you to set search criteria based on any of the attributes associated with an issue.
  • custom -
  • saved -

Advanced queries

The advanced query form allows you to define fine grained search criteria. Using this form, select the values to search through the modules artifact types issues. Searches can contain the following items:

  • Attributes - you can search on any attributes within the artifact type by entering a value.
  • Personnel - you can use this area to search for all issues associated with someone.
  • State Change - you can monitor issues that have an altered state in one attribute. This field allows you to set an attribute you would like to montior and the original value and expected new value. For instance, you may want to know all issues that have changed from a low severity to a high severity. Additionally, you can monitor these changes within a date range. For example, you may be interested in knowing which issues have received more than 2 votes within the last week. Clicking the "Update options" button saves your changes to the query.

You can also set your results view preferences at this time. You can alter the search order and the number of issues that appear per page.

You can submit your query at this time or choose the save it.

Saved queries

Once you've used the advanced query form to define your query you can add the query to your Saved query list. click the save query button and enter values on the save query form.

Query name: the query name needs to be short and descriptive. For a query that searches for all issues assigned to you, you may set the query name to "My issues". A query that searches for all issues found on the NT operating system that are resolved low priority issues, you may set the query name to "NT, resolved, low priority"
