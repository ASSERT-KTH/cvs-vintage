Issue Entry

Selecting an issue (artifact) type

Artifacts refer to the categories of issues available in your current module. Be careful to assign your issue to the appropriate artifact category. The default types are defect, patch, task, feature, and enhancements.
  • Defects are issues identified within existing feature/functionality.
  • Patches are edits submitted as fixes to defects.
  • Tasks describe undertakings required for the support of a Feature or enhancement. Tasks can be upgrading software to support new functionality, reviewing similar projects to discover how similar problems were solved, or reading up on a new technology.
  • Features are new functionality to be incorporated into the project
  • Enhancements are improvements to existing features.

Once you have selected the artifact type that best describes your issue, click on the name of the artifact. The Enter issue screen will appear.

Entering an issue, long format

Entering an issue is a three step process.

  1. Enter brief descriptive information about your issue
  2. Check the database for duplicates of your issue.
  3. Enter a full description and save the issue to the database.

Step 1: Enter descriptive information.

  • Summary: a brief statement describing the issue. Considering using keywords in this field.
  • Platform: select the affected platform from the drop down list.
  • Operating system: select the affected platform from the drop down list.

Click the "Check for duplicates" button to continue.

Step2: Check for duplicates

Once you've clicked the "Check for duplicates" button the database is scanned for similar issues how deeply in depth should we go with the process by which this is done?

If duplicates are found a report of similar issues will be presented with your issue listed at the top. Review the issues listed for fit. You can do a number of things at this point:

  • Review individual issues by clicking on their id. The full issue description will appear.
  • Sort issues by each of the categories. Click on the name of the category to change the sort order
  • Alter one or more issues at the same time. Click on the check box to the left of the issue ids that you would like to manipulate. You can then enter a note in the note field and click the submit note only button to add a note to the comments section of all issues selected. Additionally, you can add a vote to the issues you've selected by clicking the add vote to selected button.
  • Skip this process by clicking on the bypass duplicates button.

Step 3: Enter a full description - Verbose form

The Verbose Info screen allows you to add detailed information about your issue. Required fields have an * (astrick) beside their title. You should be as descriptive as possible. Use all of the attributes to describe your issue.

Setting dependencies

Issues often have dependencis on other issues. One issue may require a resolution from another before any work can be done. Two issues may have parellel resolutions and need to be tracked together, though they have no direct influence on one another. Dependencies are defined as parent/child relationships within Scarab. Child issues are those with defined dependencies on other issues.

There are three levels of dependencies within Scarab: blocking ,duplicating and non-blocking. Blocking issues prevent the issue being blocked, from being resolved. The blocking issue must be addressed before the blocked issue can be resolved.

Sometime duplicate issues may be entered and need to be tracked seperately. Marking an issue as a duplicate allows users to track efforts on both issues. Duplicates may be in different modules or the same module.

Marking issues as non-blocking dependencies also allows users to simultaneously track efforts on two or more issues.

You can track the progress of interrelated issues by using the Dependencies section of the issue and clicking on either the issue id or the Issue summary links to the issues. In this area you can also redefine the Dependency type.

Issue attachments

You can include an attachment for the issue. To do so, enter a brief description of the attachement and use the browse button to navigate to the file on your local system or enter a path for the file.
