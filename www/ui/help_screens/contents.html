Customizing "enter issue" form

Editing "Enter issue" form:

User selects "Customize / Create 'Enter Issue'": This page consists of a multiple tables ("attribute groups") listing the form elements (attributes) that appear in the enter issue form. There is a one-to-one correspondence between the naming/sequence of attribute groups (and the attributes within them) on the "Customize / Create 'Enter Issue'" screen, and the "enter issue" screen.

top 

Attribute groups:

Attribute groups can be used as a means of grouping functionally/contextually related attributes on the "Customize / Create 'Enter Issue'" screen.

Attribute groups are useful for:
  1. Grouping functionally related attributes
  2. Dividing large numbers of attributes into logical chunks in order to improve the visual scannability of the screen
  3. Creating cognitive/contextual groupings (if you understand one attribute in a group, your half way to understanding all of the attributes in a group)
top 

To create an attribute group:
  1. Click "add group" (submit @ bottom of page) this will create a new empty attribute group at the bottom of the page (default) or immediately beneath the last selected attribute group
  2. Enter a name for the new attribute group (save required)
top 

To delete an attribute group:
  1. Select an attribute group by clicking the checkbox to the left of the attribute group's name
  2. Click "delete group" (submit @ bottom of page)
top 

Managing attribute groups:
  1. Checkbox to the left of the attribute group name that allows you to show or hide a group and all attributes appearing in it
  2. Relative sequence of groups can be modified by selecting a new sequence number from the "sequence #" pull-down menu of a given attribute group
  3. Select which attribute groups are checked against the database for duplicates
top 

Managing attributes:
  1. Edit an attribute by clicking on the attribute name - this links to it's corresponding attribute "Add / Edit Attribute" screen
  2. Remove an attribute(s) by selecting the desired attribute(s) and clicking the "delete selected" submit button attached to their attribute group
  3. Rename an attribute within the project (this is desirable if a user is utilizing attributes from the "global attributes list") by entering text into the corresponding "name/project" field
  4. Checking or unchecking "activate" will show / hide attribute in the "Customize / Create 'Enter Issue'" form without deleting it
  5. The "required" checkbox assigns an attribute as a required field on the "Customize / Create 'Enter Issue'" (leaving a required attribute field blank results in an error refresh that indicates to the user what information is missing)
  6. Resort attributes within a group by selecting a new sequence number from the "sequence #" pull-down menu of a given attribute
top 

To add pre-defined attributes to an attribute group:
  1. Click the "add attribute" button appearing associated with a specific attribute group to access "global attributes"
  2. Select a pre-defined attribute(s) from "global attributes" list and click "add selected entry to page"
  3. Selected attribute(s) will appear at the bottom of the selected attribute group (default) or immediately beneath the last selected attribute in that group

To edit an attribute:
  1. Two points of entry into this process-
    1. Select attribute from customize / create "enter issue" form ("name" appears as a link to a given attributes corresponding edit attribute screen)
    2. Select an attribute from "global attributes" list ("name" appears as a link to a given attributes corresponding edit attribute screen)
  2. "Input type"-determines the what type of form element (text box, text area, select menu, pull-down menu, etc.) that the attribute will be. altering this setting will change the attribute in different ways depending on its initial conditions:
    1. Changing an attribute from a pull-down menu with list options to a text box/text area- list options hidden but not deleted
    2. Changing an attribute from a pull-down menu with list options to a select list- list options map to options in select list
    3. Changing an attribute from a pull-down menu with list options to a radio button set- list options map to radio button choices
  3. Attribute name appears as the title of the form element on the "enter issue" form
  4. Entering an attribute description will make the attribute name an anchor link to a corresponding entry on Scarab's nomenclature glossary. It will also appear as "alt" text for this link on browsers that support this feature
  5. List options (i.e. pull-downs, select list, radio buttons) can be created, edited, sorted, deleted, using the submit buttons at the bottom of the "attribute options" table.
    1. "Add option" adds an option to the bottom of the list or immediately beneath the last selected option in the list

Entering an issue

Every screen in Scarab includes a link at the top of the page allowing users access to the "enter issue" process. This process is broken into a least two steps:

  1. User completes necessary fields, entering information they have accumulated regarding the issue
  2. The system takes this information and maps it to the existing issue database looking for possible duplicates

Default settings for scarab divide the process into three steps:

top 

  1. Screen one
    1. User completes preliminary screen of fields, primarily consisting of pull-down menus and short text fields
    2. The system takes this information and maps it to the existing issue database looking for possible duplicates
    3. User enters verbose information, entering narrative text describing specifics of what makes issue unique, etc.
  2. Screen two "Identified as duplicate" functionality:
    1. First list table on screen reitterates data entered into "new issue"
    2. Second list table on screen lists out all possible duplicates of this issue, in order or relevance
    3. If the user identifies an issue(s) in this second list table that matches the issue they have entered, they can choose to either:
      1. "Add vote to selected": select the issue(s) that matches, increase the vote increment, and add a comment to each issue (this appears in the "comments" section of each selected issue)
      2. "Submit comment": add a comment to selected issue(s) without increasing the vote increment
      3. "Edit new issue""return to the previous page and edit the new issue (and thereby repeat the duplicate checking process)
      4. ignore all duplicates and proceed with entering verbose data for the new issue
  3. Screen three
    1. Entering url(s): users can add multiple URLs to an issue by clicking the "add URL" submit button, entering the "URL" and a text description of this "url". New entered URLs will be saved to the issue when any submit button is clicked (e.g. "add url", "delete url", "add/edit attachment", "delete attachment") or when the "submit issue" submit button is clicked. URLs can be deleted from this screen by first selecting the URL with it's "select" checkbox, and then clicking "delete URL".
    2. Adding an attachment(s): users can add multiple attachments to an issue by clicking the "add/edit attachment" submit button, browsing for and uploading the attachment and entering a text description. Attachments can be deleted from this screen by first selecting the attachment with it's "select" checkbox, and then clicking "delete attachment".
    3. "Workflow options": after completing data entry into the "enter issue" form (this is screen three in the default "enter issue" configuration), users can select from a radio button set at the bottom of the page that allow them to:
      1. Automatically return to the first page of "enter issue" in order to enter another issue
      2. Return to the project homepage
      3. Proceed on to the process of assigning the issue

Querying the issue database

Queries perform numerous functions within Scarab:

  1. Locate every occurence of a numerical or text string (e.g. simple search)
  2. Compare columnar data (e.g. customizable reporting)
  3. Establish/prioritize workflow (e.g. date / priority / release)
  4. Facilitate the tracking of rate-of-change (e.g. customizable reporting)
  5. Tracking open/closed issues (e.g. saved queries)
  6. Sharing information across a team (e.g. alert emails, and issue queues)
  7. Tracking user statistics (e.g. custom issue queues)
top 

Access to simple free-text query functionality appears as a textbox in the SourceCast query module:

  1. by default, users can perform a general query by entering a search string, and clicking submit
  2. the pull-down menu options allow users to refine their search, limiting it to:
    1. Issue #
    2. Date range (xx/xx-xx/xx)
    3. Searching open text fields only
    4. Custom (as determined by admin)
top 

Advanced query form maps one-to-one against the issue data fields of a project, laying out form fields in with the same nomenclature and grouping as seen on the "enter issue" page. Users define the specificity of a search by entering/selecting search strings into as many or as few form fields as necessary. This page also gives them the choice of saving an entered search as a search template. A search template can be saved to a user's account, or, with administrative access/approval, made available within an entire project.

To complete an advanced query:

  1. Access advanced query from the left hand navigation of Scarab or through the SourceCast query module
  2. Text entered into a textbox or text area will only query against that field
  3. Making selections from pull-down menus / select-lists limits query results to the selected fields
  4. Boolean searches can be added to an advance query using the following series of form fields:
    1. Boolean search type is selected from the first pull-down menu.
    2. Select a definition/limitation string from the second pull-down menu
    3. Enter a text string
    4. Click "and" or "or" to add another string of text (both of these submit buttons will refresh the screen, adding another textbox immediately below the first, preceded by "and" or "or" respectively)
    5. Clicking "add additional boolean search" will add a new, distinct instance of boolean search
  5. query results default sort value can be reset by selecting from the "sort by" menu
  6. select the number of results per page (default is 25)
  7. click "save this query" enter a name, description, and destination (to personal profile, or to project), if you want to save the search for repeat use
  8. after completing the form, or after saving a search, click "submit"

Custom query allows users to use a single text string and query against selectd text fields:

Access advanced query from the left hand navigation of Scarab or through the SourceCast query module

  1. Text search
  2. Select the attributes to be included in a text search
  3. Enter a query text string
  4. Additionally, users may further refine this query by selecting options from single option(pull-down) and or multiple option(select-list) attributes
  5. Select the number of results per page (default is 25)
  6. After completing the form, or after saving a search, click "submit"
top 

Query results appear in a similar list table format to most other instance based data in Scarab. Screen level functionality enables the following:

  1. Clicking through multiple screens of query results using "previous" and "next"
  2. Changes to any of the following remain as the default setting until modified:
    1. Selecting which queried attributes appear in the list view as column headings (e.g. if you are querying P1s exclusively, it would conserve horizontal space to exclude this column from view)
    2. Clicking on a column header will resort the query results by that attribute
    3. Query results can be sorted in ascending or descending order by clicking on the up/down arrows in the upper left-hand corner of the results list
    4. User selects the number of issues viewed per page by selecting from the radio button set at the bottom of the page

Saving/Running saved queries

  1. User's have the option of saving a query to their personal profile, or submit it for approval for use by and entire project team
  2. The save location is fixed - current project; saving to other projects opens up a large number of issues (similar to move/copy issue)
  3. Users can access query details from "Saved query list (2.2.2)(M2) and edit them dependant on their level of permission
  4. User executes query from a pulldown menu in Scarab's secondary navigation, or by clicking on that querie's "location" field on the Saved query list
