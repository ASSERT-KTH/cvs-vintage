Scarab Release 1.0
Feature/Functionality Matrix

Draft 0.2
4-13-01

FEATURE CATEGORY / Feature FEATURE DESCRIPTION
Create an Issue  

Create an Issue

 User enters an issue, duplicates are avoided if possible. The static fields of an issue are entered first along with a short description. This is used to look for duplicate issues. If it is a duplicate, the currently selected issue gains a vote. If a new issue is entered, it is well specified and actionable.

Create an Issue Based on a Template

 User creates an issue based on a previously saved or predetermined template.

Create a Template for Later Use

 User creates a template that can be employed when entering future issues.
Search/Query an Issue(s)  

Search for Issue by ID

 User enters issue number and sees the specified issue or an appropriate message.

Quick Search for Issue

 User enters a few terms and sees a list of issues that are likely to be relevant. Next, the user can drill into a specific issue from the list.

Advanced Search for Issue

 User enters detailed search criteria and sees a list of issues that match. Next, the user can drill into a specific issue from the list.
Edit an Issue

Change an Issue(s)

 User can make a number or edits to an existing issue. Supports the ability to make changes to several issues at once

Edit Milestone Details

 User can drill into a milestone object and make a change, affecting every issue with that milestone.

Defering Issues

 User can indicate that an issue is not invalid yet the development team will not work on it any time soon.

Reject an Invalid Issue

 In the course of reviewing new issues, user can reject those that are invalid as well as provide concise rationale.

Copy an Issue

 User can select an issue within a project and copy it within the same project or to other projects that they have access to. To copy within the same project, an edit would be necessary to prevent entering a duplicate issue. For a project-to-project(s) copy, this can be a straight copy with no edits necessary. In either case, a new Issue ID is created

Move an Issue

 User can move an issue from one project to another. The original issue (Issue ID) would be closed out and a new Issue ID would be created by the move. The new Issue ID would have a link back to the orginal issue.
Assign an Issue  

Assign an Issue(s)

 If an issue has been entered and was not assigned to a specific developer at that time, the component owner must assign it to a specific developer.
Reports / Metrics  

Predefined Reports

 User can select a predefined report within their project and then view and/or print that report. The predefined report would be similar to a "canned" report: defined during project set up and/or a proposed report by a user that would be approved by the Project Owner.

MyReports

 User can create a MyReports report from a query and have the report available to them across all of their projects.

Automation/Batching

 User can automate a Predefined and/or MyReport to route or print at a specific time, date and be sent to a specific email address.
Administration  

Define Issue Status

 Allow the DomainAdmin or ProjectOwner to define the issue status across projects (domain) or within a specific project

Define Status Transition Model

 Allow the DomainAdmin or ProjectOwner to define the status transition model at the global (domain) or project level

Define Attributes

 Allow the DomainAdmin or ProjectOwner to define or actually create the attributes. User would be able to either add, edit or delete exisiting attributes.

Customize Attribute Options

 Allow the DomainAdmin or ProjectOwner to customize attribute options as well as default option values. This can be accomplished at a global or project level, respectively.

Other Customization

Thresholds: Allow the DomainAdmin or ProjectOwner to set thresholds at a global and/or project level. Overall goal is to create urgency and allow stricter follow up on defined inactivity.

Email Notifications: Allow the DomainAdmin or ProjectOwner to customize email notification fields so that all future email notifcations contain the defined fields.
File Activities  

Import a Database File

 Allows user to import an XML file (database) into a project. The user can easily transfer an issue database across projects in an XML format.

Export a Database File

 Allow user to export an XML file (database) from a project. User should be able to export an entire database or via a query export only a portion of a database.

Attach a File to an Issue

 Allows user to have the option of attaching one or more files to an Issue during the creation of an issue or the editing of an existing issue. User would be able to browse their local file system or enter the name of a file attachment:
Audit Log Ability for Administrator or Project Owner to track all significant issue-related user events such as status changes, priority changes

Command-line Execution

 User can perform command line execution of common functions
Integration  

Link with Version Control

 User establishes a link between an Issue ID and the SCM (i.e. CVS) tool's Commit ID within the same project. Gives user full visibility to an issue when they attempt to close an Issue. Integrates the Issue Tracker and the SCM tool.
Security User's activities predicated by both role and permission. Roles are a collection of one or more permissions. Thus, you can either validate access based on their role or their permission
Documentation Online Help documentation.
