Scarab: Bug-Tracking Built for the Ages
The goal of the Scarab project is to build a bug tracking system that has the following features:
  • A full feature set similar to those found in other bug tracking systems: data entry, queries, reports, notifications to interested parties, collaborative accumulation of comments, dependency tracking, collaborative prioritization (bug voting).
  • Built as a set of Java servlets for speed, scalability, maintainability, and ease of installation.
  • Compatible with Bugzilla and other widely used bug tracking system. Possibly via a bug report DTD.
  • Modular design that allows manageable integration of new features over time.
  • Fully customizable through a set of administrative pages.
Although the final product will have a strong feature set, many of these features will be based on support in an underlying library of collaboration components. For example, Scarab will not implement its own notification system or localization system, that should be a reusable component of the underlying framework.
