Scarab - Issues

As of 7-9-2002, Scarab is now self hosting!

Scarab is now used to track Scarab issues. Previously, we were using the Tigris.org IssueZilla issue tracker. Do not submit new issues into Issuezilla. Please note that we have not finished the migration from the old system to the new system quite yet, so please be patient.

If you would like to submit an issue, please check the KNOWN_ISSUES.txt file to make sure that we don't already know about an issue that may not necessarily have a particular issue the problem is reported against.

After searching the issue tracker and you really do have an original problem to report, then please report it using our new issue tracker. Register and create a new account. Make sure to request a DEVELOPER role. Please include a detailed message that describes the problem (make it as easy as possible to understand), how to reproduce the problem and a stack trace if one is generated.

NOTE: Please do not use the new issue tracker to create test issues. You can use our runbox to create as many test issues as you want.

If the new issue tracker is unavailable or you have a problem with it, please contact Daniel Rall and let him know.
