* There is a new bit of terminology that I'm going to start referring to. It is called Model 2+1. It is a +1 on the Model 2 development approach that was discussed in this paper:

<http://www.javaworld.com/javaworld/jw-12-1999/jw-12-ssj-jspmvc.html>

I see Turbine fitting right into this model, but also expanding on it quite a bit to provide further support for defining things like a consistent methodology for handling form submission (actions), navigation, screens, etc as well as supporting more than one template language in a consistent manner. Hence, Turbine is now Model 2+1. :-)
