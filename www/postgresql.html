Scarab - Postgresql

$Id: postgresql.html,v 1.1 2002/02/26 22:24:01 jon Exp $

Scarab has been developed with database independence in mind, however our primary goal has been to make Scarab work with MySQL. Therefore, our support for other RDBM's is largely untested. We are relying on the user community in order to help us test and develop further support for other databases. This document outlines the steps that we think are required to get Scarab working with Postgresql. Community feedback on this document will help improve things for others.

NOTE: Scarab requires Postgresql version 7.2 and higher.

NOTE: Scarab does not currently work with Postgresql. There is a stack trace thrown when entering a new issue. We are working as quickly as possible to resolve the problem.

In order to get Torque to generate the right Scarab .sql files for Postgresql and to allow Scarab to connect to Postgres, please try the following directions:

  1. Create a scarab/build/build.properties file.
  2. In the build.properties file, fill out the following properties with the right information pertaining to Postgresql (see below for examples). Then, rm the target directory and rebuild.
  3. One should also follow the directions for setting up Postgresql to accept TCP/IP connections from localhost.

scarab.database.name=scarab
scarab.database.port=5432
scarab.database.host=localhost
scarab.database.url=jdbc:postgresql://${scarab.database.host}:${scarab.database.port}/${scarab.database.name}
scarab.database.admin.url=jdbc:postgresql://${scarab.database.host}:${scarab.database.port}/
scarab.database.type=postgresql
scarab.database.jdbc.driver=org.postgresql.Driver
scarab.database.username=[INSERT_USERNAME_HERE]
scarab.database.password=
