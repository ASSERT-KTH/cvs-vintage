Integration Use Cases

Name: UC-16: Integrating an Issue ID with a SCM Commit ID Within the Same Project
Extends: UC-09 in cases of Issue State changes.
Purpose: User establishes a link between an Issue ID and the SCM (i.e. CVS) tool's Commit ID within the same project. Gives user full visibility to an issue when they attempt to close an Issue. Integrates the Issue Tracker and the SCM tool.
Feature(s): FeatureName
Direct Actors: User: Any Role that can close an Issue.
Scarab: The scarab system.
SCM Tool: The SCM system
Stakeholders: QA Manager: wants to make sure an issue can be closed.
Project Owner: Visibilty into Issues and their respective Commits.
.
Prereq: Project set up
Issue Type=Defect and Issue State=Fixed
Results: When a Defect Issue's state is changed to "Fixed," user has the option to match the Issue ID to a Commit ID.
Data sets:
Notes/Questions:
  • Does it make sense to have Scarab attempt to link to a Commit ID only when an Issue's State is changed to "Fixed?"
  • Scarab only to check for Commit ID's within the same project. Potentially cross-projects at a later date.
  • IssueChangedConfirmation page will need to have different messages depending on whether the user votes for a Commit match or does not vote for a Commit Match
Main Success Scenario:
  1. User: visit IssuesView for a given project
  2. User: changes Issue State to "Fixed" and clicks Submit
  3. Scarab: checks for potential matching SCM Commit ID's within the same project
  4. If matches found:
    1. Scarab: display IntegrateView
    2. User: review list of Commit matches
    3. If Commit matches user's issue:
      • User: vote for matching issue
      • Scarab: links the Issue ID to the Commit ID
      • Scarab: display ProjectIssues page with IssueChangedConfirmation
    4. If no match:
      • User: indicate no match
      • Scarab: display ProjectIssues page with IssueChangedConfirmation
Alternative Scenario Extensions:
  • None
