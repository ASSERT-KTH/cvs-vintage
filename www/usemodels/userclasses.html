Scarab User Classes

Format of user classes

Name: U-01: Anonymous Guest
Extends: None
Key Requirements: Search and browse issues.
Find out if a problem they hit is a known issue.
Occasionally understand project activity and progress.
Interaction Profile: Low Frequency. Low Intensity.
Notes/Questions:
  • No contact information
  • emphasize learnability


Name: U-02: Observer
Extends: Anonymous Guest
Key Requirements: Enter issues
Track issues
Interaction Profile: Low Frequency. Low Intensity.
Notes/Questions:
  • Has real account with role in project and contact info.


Name: U-03: Developer
Extends: Observer
Key Requirements: Enter issues
Track issues they entered
Have issues assigned to them
Track issues assigned to them
Manage dependencies
Change issues
Interaction Profile: Med Frequency. Med Intensity.
Notes/Questions:
  • None


Name: U-04: Lead Developer
Extends: Developer
Key Requirements: Track team issues by milestone
Assign/reassign issues to developers and other team members.
Frequently understand project activity and progress.
Change many issues at once as they relate do larger project plans.
Interaction Profile: Med Frequency. Med Intensity.
Notes/Questions:
  • None


Name: U-05: Project Owner
Extends: Developer
Key Requirements: Assign/reassign issues to developers and other project members.
Track all issues by milestone.
Constantly understand project activity and progress.
Produce reports for off-line stakeholders.
Change many issues at once as they relate do larger project plans.
Reconfigure the tool.
Interaction Profile: High Frequency. High Intensity.
Notes/Questions:
  • None


Name: U-06: QA Engineer
Extends: Observer
Key Requirements: Enter issues
Track issues they entered
Have issues assigned to them
Track issues assigned to them
Manage dependencies
Change issues
Relate issues to test plans
Verify that resolved issues are really resolved
Generate new test cases to regression test closed issues
Interaction Profile: High Frequency. High Intensity.
Notes/Questions:
  • None


Name: U-07: QA Manager
Extends: Project Owner
Key Requirements: Basically the same as project owner, but more day-to-day usage and fewer unilateral actions.
Interaction Profile: High Frequency. High Intensity.
Notes/Questions:
  • None


Name: U-08: Domain Admin
Extends: None
Key Requirements: Some domain-wide admin of the scarab tool.
Support end users when documentation is not enough.
Interaction Profile: Low Frequency. Low Intensity.
Notes/Questions:
  • Does not work directly with the tool much.


Name: U-09: Consulant/Contractor
Extends: Developer
Key Requirements: Enter issues
Track issues they entered
Have issues assigned to them
Track issues assigned to them
Manage dependencies
Change issues
Interaction Profile: Med Frequency. Med Intensity.
Notes/Questions:
  • Information visibility with system limited by the terms of an NDA. Works more closely with regular developers.


Name: U-03: Partner Employee
Extends: Developer
Key Requirements: Track only the issues they entered
Have issues assigned to them
Track issues assigned to them
Manage dependencies
Change issues they entered
Interaction Profile: Med Frequency.
Notes/Questions:
  • Information visibility with system highly limited by the terms of an NDA. Does not work closely with regular developers.
