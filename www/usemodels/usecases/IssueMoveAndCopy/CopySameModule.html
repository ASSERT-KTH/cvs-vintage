Primary Actor: Any user
Goal in context: Copy one or more artifacts to a different artifact type within the same project.
Level: User goal
Scope: Single project
Preconditions: User has Project Tracking - Enter Issue permission in project.
All source artifacts are of the same artifact type.
Success guarantees: The following are duplicated in target artifact: shared attributes, Artifact ID, creation date, created by, history, comments, attachments, dependencies, URLs.
Values of unshared attributes in source artifact are entered as an additional comment in the target artifact or deleted.
Trigger event: User selects one or more artifacts to be copied.
Main Success Scenario: 1. User selects target artifact type.
2. System presents mapped attribute values from source to target.
3. User confirms mapping.
4. System changes artifact type.
Scenario Extensions: 1a. No target artifact type selected:
  1a1. Request target artifact type. Return to step 1.
3a. No reason for change entered:
  3a1. Request reason for change and return to step 4.
3b. Attribute exists and is set in source and does exist not target:
  3a1. Attribute is highlighted as not existing in target.
  3a1. User selects one or more highlighted attributes to appear in target comment.
  3a2. Use case continues.
* Permissions change:
*.1 System informs user of permission requirement.
*.2 Use case ends.
Technology & Variations List: None
Notes and Issues: Only one target artifact type may be selected.
