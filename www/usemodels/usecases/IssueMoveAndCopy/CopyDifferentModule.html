Primary Actor: Any user
Goal in context: Copy one or more artifacts to different module.
Level: User goal
Scope: Cross-project
Preconditions: User has Project Tracking - View Issue permission in source project.
User has Project Tracking - Enter Issue permission in at least one target project.
All source artifacts are of the same artifact type.
Artifact type is defined in target project.
Success guarantees: Source artifact is annotated to indicate it has been copied.
All source artifact attributes remain unchanged.
Artifact copy exists in the target.
The following are duplicated in target artifact: shared attributes, artifact ID, creation date, created by, history, comments, attachments, dependencies, URLs.
All other attributes have been mapped to target artifact attributes or entered as a comment in the target artifact.
Users associated to source artifact without required target module permissions are noted as a comment in target artifact.
Users associated to source artifact with required target module permissions are associated appropriately.
Trigger event: User selects one or more artifacts to be copied.
Main Success Scenario: 1. User selects target project and artifact type.
2. System presents mapped attribute values from source to target.
3. User confirms mapping.
4. System copies artifact(s).
Scenario Extensions: 1a. No target project and artifact type selected:
  1a1. Request target project and artifact type. Return to step 1.
3a. No reason for change entered:
  3a1. Request reason for change and return to step 4.
3b. Attribute exists and is set in source and does exist not target:
  3a1. Attribute is highlighted as not existing in target.
  3a1. User selects one or more highlighted attributes to appear in target comment.
  3a2. Use case continues.
* Permissions change:
*.1 System informs user of permission requirement.
*.2 Use case ends.
Technology & Variations List: None
Notes and Issues: Only one target project and artifact type pair may be selected.
