Name: UC-04 - Advanced search for Issues
Extends: None
Description: User enters detailed search criteria and sees a list of Issues that match. Next, the user can drill into a specific Issue from the list.
Feature(s): None
Direct Actors:
  • User : Any Role (High Priority, High Frequency)
  • Scarab : The scarab system
Stakeholders:
  • Project Participant : Wants to be able to search for an Issue with great detail (including all Attribute fields) and quickly find it.
PreRequirements:
  • Logged in
  • Project set up
  • Existing Issue
  • User knows some search terms
Results:
  • A list of matching Issues are displayed or go directly to the Issue if only one matches.
DataSets:
  • Query Terms
Q/A:
  • If all Issues are in project context, is there a quick way to jump to Issue in another project?
    • If possible a link to the other project should be provided.
  • How can this functionality be put on tigris StartPage?
    • As a small text entry box.
Notes:
  • Need to consider the ability to save/restore searches. This functionality should go into a search editor usecase.
Senario:
  1. User: click 'Advanced Search' link
  2. Scarab: display AdvancedSearch page
  3. User: Enter search criteria into field and hits 'search' button. There may be other buttons on the page to deal with AND/OR style searching similar to Bugzilla's functionality.
  4. [validation] Scarab: validates search criteria
  5. Scarab: searches for matching issues
  6. Scarab: filters search based on users role permissions.
  7. if results.size( ) == 0:
    • Scarab: display message on ProjectIssues with field data as entered
    • User: Modify search criteria and repeat
  8. if results.size( ) == 1:
    • Scarab: display issue view page
  9. if results.size( ) > 1:
    • Scarab: display issue list page
    • User: survey resulting list
    • User: select one of the presented issues
    • User: sort issues based on column heading
    • User: modify displayed column headings
