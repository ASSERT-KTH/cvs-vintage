Name: UC-02 - Search for Issue by ID
Extends: None
Description: User enters Issue ID and sees the details of the specified Issue or an appropriate message.
Feature(s): None
Direct Actors:
  • User : Any Role (High Priority, High Frequency)
  • Scarab : The scarab system
Stakeholders:
  • Project Participant : wants to be able to specify issue unamgiously via an easy-to-type-or-say string. This is often done via email, in code comments, in chat, or verbally on phone or in meetings. Challenge is that unambiguous specification is at odds with brevity. Assumes project name or URL as shared knowledge.
PreRequirements:
  • Logged in
  • Project set up
  • Exiting Issue
  • User knows Issue ID
Result: Details of the Issue is displayed
DataSets:
  • Issue ID
Q/A:
  • If all Issues are in project context, is there a quick way to jump to Issue in another project?
    • If possible a link to the other project should be provided.
  • How can this functionality be put on tigris StartPage?
    • As a small text entry box.
Notes:
  • If validation fails, Scarab displays an error
Senario:
  1. User: visit ProjectIssues, or any Scarab page in project context.
  2. User: Enter issue ID into small field and hit button
  3. [validation] Scarab: validates issue number
  4. [validation] Scarab: validates user role permissions
  5. Scarab: display IssueView details
