Name: UC-06 - Run Saved Issue query
Extends: UC-05
Description: User is given the ability to run a previously staved query.
Feature(s): None
Direct Actors:
  • User : Any Role (High Priority, High Frequency)
  • Scarab : The scarab system
Stakeholders:
  • Project Participant : Wants to be able to search for an Issue with great detail (including all Attribute fields) and then be able to save it for reuse at a later date.
PreRequirements:
  • Logged in
  • Project set up
  • Existing Issue
  • Saved Query Exists
Results:
  • User clicks on saved query. Result page is the Issue view or the Issue list page.
DataSets:
  • Query Terms
  • Saved Query Name
Q/A:
Notes:
  • Need to consider the ability to modify saved searches. This functionality should go into a search editor usecase.
Senario:
  1. User: click saved search link
  2. Scarab: display Search results
  3. [validation] Scarab: validates search criteria
  4. Scarab: searches for matching issues
  5. Scarab: filters search based on users role permissions.
  6. if results.size( ) == 0:
    • Scarab: display message on ProjectIssues with field data as entered
    • User: Modify search criteria and repeat
  7. if results.size( ) == 1:
    • Scarab: display issue view page
    • User: enter name of search to save. Clicks save button
  8. if results.size( ) > 1:
    • Scarab: display issue list page
    • User: survey resulting list
    • User: select one of the presented issues
    • User: sort issues based on column heading
    • User: modify displayed column headings
    • User: enter name of search to save. Clicks save button
