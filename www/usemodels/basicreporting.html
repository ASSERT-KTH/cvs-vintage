Basic Reporting Use Cases

Name: UC-18: User Views and Prints a Predefined Report
Extends: None
Purpose: User can select a predefined report within their project and then view and/or print that report. The predefined report would be similar to a "canned" report: defined during project set up and/or a proposed report by a user that would be approved by the Project Owner.
Feature(s): FeatureName
Direct Actors: CompOwner: Component Owner (High Priority, High Frequency)
Scarab: The scarab system.
Stakeholders: CompOwner:
Prereq: Logged In
Project set up
User has access to the project
Report has already been predefined or approved
Results: User views and/or prints a predefined report
Data sets:  
Notes/Questions: Does it make sense to have a seperate Reports Page? Maybe a link in SourceCast on Developer Homepage?
Reports link from ProjectIssues page to a Reports Page?
No anonymous users can view and print reports
Predefined reports are project specific? Project Owner would define these during project creation?
Should Scarab display a Print Confirmation window or message or just print?
Main Success Scenario:
  1. User: visit ProjectIssues for a given project. Selects Reports page link
  2. Scarab: display Reports Page
  3. CompOwner: selects either one or several of available predefined reports.
  4. Scarab: Displays selected predefined report(s)
  5. CompOwner: views selected report and clicks "Print"
  6. Scarab: display a Print Confirmation window/message
  7. CompOwner: confirms Print Confirmation
  8. Scarab: prints report(s) to user's printer
Alternative Scenario Extensions:
  • None
