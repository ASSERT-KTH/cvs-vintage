Basic Reporting Use Cases

Name: UC-18: User Views and Prints a Predefined Report
Extends: None
Purpose: User can select a predefined report within their project and then view and/or print that report. The predefined report would be similar to a "canned" report: defined during project set up and/or a proposed report by a user that would be approved by the Project Owner.
Feature(s): FeatureName
Direct Actors: CompOwner: Component Owner (High Priority, High Frequency)
Scarab: The scarab system.
Stakeholders: CompOwner:
Prereq: Logged In
Project set up
User has access to the project
Report has already been predefined or approved
Results: User views and/or prints a predefined report
Data sets:  
Notes/Questions: Does it make sense to have a seperate Reports Page? Maybe a link in SourceCast on Developer Homepage?
Reports link from ProjectIssues page to a Reports Page?
No anonymous users can view and print reports
Predefined reports are project specific? Project Owner would define these during project creation?
Should Scarab display a Print Confirmation window or message or just print?
Main Success Scenario:
  1. User: visit ProjectIssues for a given project. Selects Reports page link
  2. Scarab: display Reports Page
  3. CompOwner: selects either one or several of available predefined reports.
  4. Scarab: Displays selected predefined report(s)
  5. CompOwner: views selected report and clicks "Print"
  6. Scarab: display a Print Confirmation window/message
  7. CompOwner: confirms Print Confirmation
  8. Scarab: prints report(s) to user's printer
Alternative Scenario Extensions:
  • None


Name: UC-19: User Creates a MyReports report
Extends: UC-05
Purpose: User can create a MyReports report from a query and have the report available to them across all of their projects.
Feature(s): FeatureName
Direct Actors: CompOwner: Component Owner (High Priority, High Frequency)
Scarab: The scarab system.
Stakeholders: CompOwner:
Prereq: Logged In
Project set up
User has access to the project
User needs to be in the IssueList to create a MyReport
Results: User saves a query as a MyReport. Report would then be displayed in the Reports Page as a MyReports
Data sets:
Notes/Questions: Allow user to save a query as a MyReport in the IssueList page
May make sense to allow user to create a report from the Reports Page as well - a link from this page to the ProjectIssues - may be a clumsy navigation?
No anonymous users can view and print reports
MyReports reports are project specific? Or should they be across all of the user's projects?
Should Scarab display a Print Confirmation window or message or just print?
Main Success Scenario:
  1. CompOwner: visit IssueList page after running a seach/query
  2. CompOwner: clicks a "Save as MyReports" button
  3. Scarab: display a dialogue box or confirmation window asking user to confirm parameters.
  4. CompOwner: confirms parameters
  5. Scarab: saves parameters and creates the saved query as a MyReports
  6. Scarab: takes user to IssueList page with a confirmation message.
Alternative Scenario Extensions:
  1. User: visit ProjectIssues for a given project. Selects Reports page link
  2. Scarab: display Reports Page
  3. CompOwner: clicks a "Create a MyReports" link
  4. Scarab: takes user to the ProjectIssues page to build a query (see notes - clumsy navigation)
  5. CompOwner: enters search parameters and clicks submit
  6. Scarab: display IssueView
  7. Same steps as above from Step 2 to Step 6

Name: UC-21: User Automates a Report
Extends: None
Purpose: User can automate a Predefined and/or MyReport to route or print at a specific time, date and be sent to a specific email address.
Feature(s): FeatureName
Direct Actors: CompOwner: Sets the parameters for report automation
Scarab: The Scarab system
Stakeholders: CompOwner
Recipient(s) of report(s)
ProjectOwner: see notes/comments
Prereq: Logged In
Project set up
User has access to the project
Report has already been created and saved by user or exists as a predefined report within the project
Results: User selects a report to automate and the report's parameters are saved.
Data sets:  
Notes/Questions: Should the Project Owner and/or Domain Admin have approval rights prior to an automated report being routed to somebody outside the project?
More of a UI design note: have a checkbox next to each report that would allow user to either View, Print or Automate
UI Design: A dialogue box or window allowing user to set automation parameters
Main Success Scenario:
  1. User: visit ProjectIssues for a given project. Selects Reports page link
  2. Scarab: display Reports Page
  3. User selects the appropriate check box next to each report that they would like to batch and/or automate
  4. User clicks a "Automate/Batch" button
  5. Scarab: display a Automate/Batch window or confirmation dialogue box asking user to set: Time, Date, and email addressees
  6. User enters information in the dialogue box
  7. Scarab: saves parameters and returns User to Reports Page with a confirmation message
Alternative Scenario Extensions:
  • None


Name: UC-22: User Requests to Change a MyReport to a Predefined Report
Extends: UC-20
Purpose: User can request to change a MyReport to a Predefined Report to be accessible to all users within the project
Feature(s): FeatureName
Direct Actors: CompOwner: sends request
Project Owner: Needs to approve the report
Project members: will have access to the report if approved
Stakeholders: CompOwner
Project Owner
All Project members
Prereq: Logged In
Project set up
User has access to the project
Report has already been created and saved by user
Results: User requests that their MyReport become a Predefined Report within the project
Data sets:  
Notes/Questions: Similar to having a saved query be considered a global query
Main Success Scenario:
  1. See UC-19 up to and including step 4..
  2. CompOwner: views selected report and clicks the checkbox next to one or several of their MyReport's
  3. CompOwner: clicks a "Save as Predefined" button
  4. Scarab: display a dialogue box or confirmation window asking user to confirm
  5. CompOwner: confirms reports to be changed
  6. Scarab: puts the report in a "pended" status and brings user back to Reports Page with a confirmation message
  7. Scarab: sends pending status to Project Owner for viewing and approval (see Admin Use Cases)
Alternative Scenario Extensions:
  • None
