Wireframes

Wireframe UI models are intended to outline the contents of UI screens. They consider issues of information content, complexity, navigation, widget selection, rough layout, and labeling. They are not committments to final or fine-grained UI choices. Wireframes should be conistent with the use cases and screen descriptions that reference them. Wireframes can be clicked through, but they do not have any programmed logic.

Wireframes contain some notes and marks that will not be in the final UI. Specifically, text inside braces is a condition on the preceeding UI element. Name of roles inside braces mean that the preceeding UI element will only be seen by people in that role. Names of milestones in braces or brakets mean that the indicated UI element will only be put in when work on that milestone begins.
