Browsing and Searching Use Cases

Name: UC-02: Search for Issue by ID (later: or Global ID)
Extends: None
Purpose: User enters issue number and sees the specified issue or an appropriate message.
Feature(s): FeatureName
Direct Actors: User: Any Role.
Scarab: The scarab system.
Other Stakeholders: Fellow Project Participant: wants to be able to specify issue unamgiously via an easy-to-type-or-say string. This is often done via email, in code comments, in chat, or verbally on phone or in meetings. Challenge is that unambiguous specification is at odds with brevity. Assumes project name or URL as shared knowledge?
Prereq: Logged in
Project set up
Results: Specified issue is displayed or error message.
Data sets: Issue number.
Notes/Questions: If all issues are in project context, is there a quick way to jump to issue in another project? How can this functionality be put on tigris StartPage?
Main Success Scenario:
  1. User: visit ProjectIssues, or any Scarab page in project context.
  2. User: Enter issue ID into small field and hit button
  3. [validation] Scarab: validates issue number
  4. Scarab: display IssueView
Alternative Scenario Extensions:
  • [validation] if validation fails, Scarab displays InvalidIssueNumber page
