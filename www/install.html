Scarab Installation

This is an initial pass at the documentation for installation. It is very thrown together rightnow and not complete, especially considering the software isn't complete yet either <smile>. Eventually, the goal is that we will just have a copy of Tomcat + Scarab that you can just download, double click on Tomcat start scripts and go.

Database

  • Create a database called "scarab".
  • Import the Turbine database scripts for your database (see turbine/docs/*.sql)
  • Import the Scarab database script (scarab-mysql.sql or scarab-oracle.sql)
Servlet Engine - Tomcat

Add the following to your Tomcat conf/server.xml file:

        <Context path="/scarab" docBase="webapps/scarab" debug="0" reloadable="true" >
        </Context>

Building Turbine and Scarab Java Source Code

Use the build scripts in the turbine/build and scarab/build directory to build .jar files. You should be able to execute build-turbine.[sh|bat] and build a turbine.jar. The same for Scarab...build-scarab.[sh|bat] and build a scarab.jar.

WAR Archive

Create a WAR archive using the Turbine WAR archive creation tool. Please see the turbine/docs/war-webmacro.html file for more information on that.

Copy that WAR archive into the Tomcat/webapps directory and name it "scarab".

Copy scarab/bin/scarab.jar to the Tomcat/webapps/scarab/WEB-INF/lib directory.

Copy scarab/src/templates to the Tomcat/webapps/scarab/WEB-INF/templates directory.

Property Files

Edit the Tomcat/webapps/scarab/WEB-INF/lib/Webmacro.properties and the Tomcat/webapps/scarab/WEB-INF/conf/TurbineResources.properties files....

In the Webmacro.properties file, specify the full path to the Tomcat/webapps/scarab/WEB-INF/templates directory. Also specify a path to a log file for logging errors.

http://www.server.com:8080/scarab/Turbine
